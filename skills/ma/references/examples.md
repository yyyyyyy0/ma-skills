# ma Examples

本文には 1 例だけを残し、追加例はここで管理する。

## Example 1: ダッシュボード再設計

Before:
- KPI カードが並んでいるだけで、何から見ればいいか分からない
- CTA が画面下に埋もれている

After:
- 先頭で「今の状態」を示す指標を固定
- 次に原因を見るセクションを置く
- 最後に次の行動を促す CTA を配置

## Example 2: 設定画面の整理

Before:
- 1 画面に高度設定まで全部出ている
- 項目が多く、関係が分からない

After:
- 基本設定を先頭に集約
- 高度設定は段階的開示に分離
- 余白と見出しでグループを明確化

## Example 3: フォームの意味整理

Before:
- ラベルが弱く、プレースホルダー依存
- 補助文が入力の前後で散らばっている

After:
- 先にラベル、次に入力、最後に補助文の順へ固定
- 主要入力だけを最初に見せ、二次入力は条件付き表示にする

## Example 4: ブランド制約で色面を強く使うランディング

Situation:
- ブランドガイドラインでヒーロー領域に強い色面と補助グラデーションが必須
- ただしコピーの優先順位と主要CTAは崩したくない

Apply `ma`:
- グラデーションはヒーロー全面の雰囲気づくりではなく、ブランド想起と一次CTAの視線集約に限定する
- CTA、価値提案、信頼要素の順で認知順序を固定し、色面はその順序を壊さない範囲で使う
- デザイン根拠では「ブランド要件を満たしつつ、どの要素を先に見せるための色か」を必ず説明する

## Example 5: グラデーションがデータの連続量を示す分析UI

Situation:
- ヒートマップでリスクスコアの強弱を連続量として見せたい
- 単色では差分が読みにくい

Apply `ma`:
- グラデーションは装飾ではなく、低リスクから高リスクへの連続量を表すルールとして使う
- 同じ色ルールを凡例、ツールチップ、詳細パネルでも一貫させる
- デザイン根拠では「何の差分を色で見せるか」「誤読を防ぐ補助情報は何か」を書く

## Example 6: elevation と hover が状態と因果を伝えるボードUI

Situation:
- カードをドラッグして優先順位を並べ替えるタスクボード
- hover で操作可能対象を示したい

Apply `ma`:
- 影は浮遊レイヤーとドラッグ中の対象だけに限定し、静止カードには使いすぎない
- hover は視覚ノイズではなく、つかめる対象・次に起こる操作を示す差分として使う
- デザイン根拠では「影がどの前後関係を示すか」「hover がどのアフォーダンスを伝えるか」を明記する
